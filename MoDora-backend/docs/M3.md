# M3：实现步骤

## 0. 目标（只做现在最需要的）
- Qwen3-VL：用 lmdeploy 部署为 OpenAI-compatible 服务，由后端入口拉起子进程。
- PPStructureV3：直接集成在现有 FastAPI 后端里，对前端提供一个结构化接口。
- 两者分 GPU：避免同卡常驻导致 OOM。
- 暂不做 embedding、暂不做多实例调度/熔断/复杂分层重构。

## 1. 最简架构
- modora-backend（FastAPI）
  - 提供前端接口：`/ppstructure/extract`（PPStructureV3）
  - 启动时拉起 lmdeploy 子进程（Qwen3-VL），并在调用前确保可用
  - 将“问答/多模态生成”请求转发到 lmdeploy 的 `/v1/chat/completions`
- lmdeploy api_server（子进程）
  - 提供：`/v1/models`、`/v1/chat/completions`（OpenAI-compatible）

## 2. GPU 分配（单机 8×4090，先跑通再优化）
推荐先固定两张卡：
- GPU0：Qwen3-VL（lmdeploy 子进程）
- GPU6：PPStructureV3（后端进程内）

两种绑定方式二选一：
- 方式 A（最省事）：启动后端时设置 `CUDA_VISIBLE_DEVICES=6`，后端里 PPStructure 统一用 `gpu:0`（因为只看到一张卡）
- 方式 B：后端不绑卡，在 PPStructure 初始化时显式指定设备（例如 `gpu:6`）

## 3. Qwen3-VL（lmdeploy）启动与验收

### 3.1 单实例启动命令
```bash
CUDA_VISIBLE_DEVICES=0 lmdeploy serve api_server <qwen_model_path_or_hf_repo> --server-port 9001
```

### 3.2 “可用/已加载”的判定
lmdeploy 不需要你额外实现 `/readyz`，最简单的 ready 判定是：
- 能成功访问：`GET http://127.0.0.1:9001/v1/models`

参考文档（OpenAI Compatible Server）：https://lmdeploy.readthedocs.io/en/latest/llm/api_server.html

### 3.3 后端转发的 base_url
后端调用时将 OpenAI client 的 `base_url` 设置为：
- `http://127.0.0.1:9001/v1`

## 4. 后端入口拉起 lmdeploy 子进程（关键步骤）

### 4.1 启动时做什么
1) 读取配置（模型路径、端口、CUDA_VISIBLE_DEVICES）
2) `subprocess.Popen(...)` 拉起 lmdeploy
3) 循环探活 `/v1/models`（例如最多等 5–10 分钟）
4) 探活成功后，后端才开始对外提供“会调用 Qwen 的接口”

### 4.2 退出时做什么
1) FastAPI shutdown 事件中 `terminate()` 子进程
2) 等待一小段时间（例如 3–10s）后仍未退出则 `kill()`

### 4.3 最小配置项（建议先用环境变量）
- `MODORA_QWEN_MODEL`: `<qwen_model_path_or_hf_repo>`
- `MODORA_QWEN_PORT`: `9001`
- `MODORA_QWEN_CUDA_VISIBLE_DEVICES`: `0`

## 5. PPStructureV3：直接做成后端接口（给前端）

### 5.1 后端提供的接口（建议）
- `POST /ppstructure/extract`

入参（最小）：
- `image_base64`: JPEG/PNG base64

出参（最小）：
- `blocks`: list，每个包含
  - `id`
  - `bbox`（[x1,y1,x2,y2]）
  - `type`（统一小写）
  - `text`

### 5.2 初始化位置（建议）
- FastAPI startup 时初始化 PPStructureV3，确保“无冷启动”

### 5.3 失败保护（先做最小的）
- 单次推理超时：后端做请求级超时或线程/进程隔离（先不强制）
- OOM：先通过固定 GPU + 控制并发避免（最简单：后端对该接口加一个全局 semaphore，限制并发=1 或 2）

## 6. 最小验收清单（先确保能跑通）
- 启动后端时自动拉起 lmdeploy，且能通过 `/v1/models` 判定可用
- 调用一次 `/ppstructure/extract` 能稳定返回 blocks
- 调用一次 Qwen 的 `/v1/chat/completions` 能稳定返回文本（必要时再验证多模态图像输入）

## 7. 后续可选优化（现在先不做）
- Qwen 多实例：每卡一进程（9001/9002/…），后端做简单轮询
- PPStructure 独立进程/服务：当后端需要更强隔离或更高吞吐时再拆
